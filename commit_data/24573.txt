    [dev.ssa] cmd/compile: respect phi values in tighten
    
    Given (say)
    
    b1: <- b2 b3
      v1 = Phi <t> v2 v3
    b2:
      v2 = ...
    b3:
      ...
    
    tighten will move v2 to b1, since it is only used in b1.
    
    This is wrong; v2 needs to be evaluated before entering b1.
    Fix it.
