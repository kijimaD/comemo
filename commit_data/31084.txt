    crypto/aes: fix wrong instruction on ppc64le implementation
    
    In the newest AES implementation in asm for ppc64le, this part
    
    MOVW $·rcon(SB), PTR
    
    should be
    
    MOVD $·rcon(SB), PTR
    
    since it is loading a doubleword value into PTR.
